# ScreenWire AI — Layer 2 CLI Skills

All scripts are standalone Python CLIs using `argparse` and `httpx`. Each outputs `SUCCESS: ...` or `ERROR: ...` to stdout. HTTP calls target `localhost:8000`. Project-relative paths are resolved against `$PROJECT_DIR` (default: `test_project/sw_test001_greenhouse-letter`).

---

## 1. sw_read_manifest

No arguments. GETs `/api/project/current` and prints a condensed summary: project name, status, phase statuses, cast/location/prop/frame counts, and frame status breakdown.

```
./sw_read_manifest
```

---

## 2. sw_queue_update

Writes a manifest micro-update JSON to `{PROJECT_DIR}/dispatch/manifest_queue/micro-update_{epoch_ms}.json`. No HTTP call.

```
./sw_queue_update --payload '{"key": "value"}'
```

| Arg | Required | Description |
|---|---|---|
| `--payload` | yes | JSON string (markdown fences stripped automatically) |

---

## 3. sw_update_state

Reads `logs/{agent}/state.json`, merges new fields, writes back (creates if missing).

```
./sw_update_state --agent voice_director --status done --sub-phase casting
```

| Arg | Required | Description |
|---|---|---|
| `--agent` | yes | Agent ID |
| `--status` | yes | New status value |
| `--sub-phase` | no | Optional sub-phase string |
| `--file` | no | Override output path |

## 4. sw_generate_image

POSTs to `/internal/generate-image`.

```
./sw_generate_image --prompt "A greenhouse at golden hour" --out assets/frames/frame_01.png
```

| Arg | Required | Default | Description |
|---|---|---|---|
| `--prompt` | yes | — | Generation prompt |
| `--out` | yes | — | Output path (relative or absolute) |
| `--size` | no | `landscape_16_9` | Image size preset |
| `--steps` | no | `28` | Inference steps |
| `--guidance` | no | `3.5` | Guidance scale |

## 6. sw_generate_video

POSTs to `/internal/generate-video`. Maps `p-video` -> `prunaai/p-video`, `grok-video` -> `xai/grok-imagine-video`.

```
./sw_generate_video --api p-video --image assets/frame_01.png --prompt "slow pan" --out render/clip_01.mp4
```

| Arg | Required | Description |
|---|---|---|
| `--api` | yes | `p-video` or `grok-video` |
| `--image` | yes | Input image path (relative or absolute) |
| `--prompt` | yes | Video generation prompt. For grok-video, spoken dialogue and delivery belong in the prompt's `AUDIO:` section. |
| `--out` | yes | Output path (relative or absolute) |
| `--audio` | no | Legacy optional audio file path. Not used by the native-audio grok-video pipeline. |
| `--duration` | no | Optional duration in seconds (int). For dialogue clips, pass the assembled prompt duration rather than defaulting to 5s. |

## 10. skill_slice_audio

Slices a time segment from an audio file using `ffmpeg`.

```
./skill_slice_audio --input audio/full.mp3 --start 2.5 --end 8.0 --out audio/slice.mp3
```

| Arg | Required | Description |
|---|---|---|
| `--input` | yes | Input audio file |
| `--start` | yes | Start time (seconds, float) |
| `--end` | yes | End time (seconds, float) |
| `--out` | yes | Output file |

---


| Arg | Required | Description |
|---|---|---|
| `--video` | yes | Input video file |
| `--out` | yes | Output image file |

## 13. skill_verify_media

Runs `ffprobe` on a media file and prints formatted metadata (duration, size, bitrate, codec, dimensions, framerate).

```
./skill_verify_media --file render/clip_01.mp4
```

| Arg | Required | Description |
|---|---|---|
| `--file` | yes | Media file to inspect |
